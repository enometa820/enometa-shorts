# 모델 전환 프로토콜

## 사용 모델
- **Sonnet**: 기본 작업 모델
- **Opus**: 복잡한 작업에만 사용
- **Haiku**: 사용하지 않음

## Sonnet 작업
- 문서/설정 파일 수정
- 단일 컴포넌트 파라미터 추가/수정
- 기존 패턴을 따르는 코드 확장
- 버그 수정, 린트, 리팩토링
- 에피소드 제작 파이프라인 실행
- 피드백 정리, 문서 최신화

## Opus 작업
- 새로운 아키텍처 설계
- 여러 파일에 걸친 구조적 변경
- 복잡한 알고리즘 구현
- 시스템 전체에 영향을 주는 의사결정
- 감사/분석 보고서 작성
- 스킬 구조 변경

## 전환 제안 시점
1. **세션 시작**: 요청 복잡도 판단 → 현재 모델 적절한지 안내
2. **작업 전환**: 단계가 끝나고 다음 단계 성격이 다를 때
3. **예상보다 복잡**: Sonnet으로 시작했는데 복잡해질 때
4. **과잉 사용**: Opus로 단순 작업 중일 때

## 제안 형식

```
> **모델 전환 제안**: [현재] → [제안]
> **이유**: [한 줄]
> **작업**: [뭘 할 건지]
```
