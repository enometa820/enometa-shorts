## 설명란 작성 가이드

- **문체**: **합니다체** 필수 (합니다/입니다/됩니다/줄입니다 등)
- **5~7문장** (기존 2~3문장에서 확장)
- 1단 (1~2문장): 대본 핵심 데이터/수치/사실 — "이게 뭔데?"를 답하는 훅
- 2단 (2~3문장): 철학적/인문학적 의미 — "그래서 우리한테 왜 중요해?"
- 3단 (1~2문장): ENOMETA 고유 관점 명시 — 5대 도메인 교차점 언급
- 구분선 아래: **고정 멘트 변경 불가**

## 고정 댓글 작성 가이드

- **문체**: **합니다체** 필수 — 인용문(따옴표 내부)은 원문 그대로 유지, 나머지 설명/맥락은 합니다체
- **1단 인용**: 대본에서 가장 강렬한 1~2줄, 따옴표(`" "`) 형식 (원문 유지)
- **2단 맥락**: 인용 문장의 의미를 2~3줄로 보완 — 알고리즘/과학/철학적 배경 한 줄 + 삶의 맥락
- **3단 질문**: 시청자가 자기 삶을 대입할 수 있는 열린 질문 1~2개
  - 너무 광범위한 질문 금지 ("어떻게 생각하세요?" 수준 X)
  - 대본의 구체적 상황에서 파생된 질문 (예: "지금 동시에 쥐고 있는 선택지가 있나요?")
- 구분선 아래: **고정 멘트 변경 불가**

## 태그 규칙

- 주제 기반 **5개만**
- **SEO 포괄적 상위 카테고리** 우선 — 구체적 전문 용어보다 검색량 높은 상위어 선택
  - 예: `#이진탐색` → `#알고리즘` / `#내려놓음` → `#자기계발` / `#정보이론` → `#철학`
  - 에피소드 주제의 5대 도메인(뇌과학/데이터분석/철학/인문학/컴퓨팅)에서 상위어 선택
- **완전 금지 태그** (어디에도 절대 사용 불가): `#쇼츠` `#shorts` `#ENOMETA` `#이노메타`
- **고정 해시태그** (항상 포함, 5개 주제 태그 뒤에 추가): `#데이터아트` `#전자음악` `#오디오비주얼` `#철학`
  → 해시태그 섹션 총 9개: 주제 5개 + 고정 4개

- 태그 5개 확인, 제외 태그 포함 여부 확인
