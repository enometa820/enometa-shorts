# ENOMETA Writing System v12 — Music-First (1 Sentence = 1 Bar)

## 0. 가장 중요한 핵심 원칙: "1문장 = 1마디"

ENOMETA의 영상은 **음악(박자) 중심**으로 생성됩니다. 대본은 산문(에세이)이 아니라 **시(Poetry)** 또는 **랩(Rap) 가사**의 형태를 띠어야 합니다.

- **[1줄 = 1마디]**: 대본에서 줄바꿈(Enter) 한 번이 음악의 1마디(약 1.77초)가 됩니다.
- **[최적 글자 수]**: 1마디에 자연스럽게 들어가는 글자 수는 **8자~12자**입니다. 
- **[가변 마디 룰]**: 
  - 극단문 (1~6자) : 남는 시간은 음악 비트가 채우는 '솔로(Solo)' 구간. (예: "공포.")
  - 표준문 (7~15자) : 1마디 꽉 차는 밀도 높은 래핑. (예: "각성의 화학식은 같다.")
  - 장문 (16자 이상) : 시스템이 자동으로 2마디로 늘려 배치함 (지양할 것).
- **[드롭 구간]**: 의도적으로 **빈 줄(Empty line)**을 넣으면, 1마디 동안 나레이션 없이 음악만 폭발하는 드롭(Drop) 구간이 됩니다.

## 1. 채널 아이덴티티

**ENS·NOĒMA·META** = 존재·사유·초월
"존재와 사유, 그 경계를 초월하다"

### "팩트로 위로하는" 철학 (Comfort Through Facts)

```
원칙: 사실은 설교하지 않는다. 사실은 드러낸다. 드러냄이 위로한다.
패턴: 과학적 사실 + 리프레임 = 설교 없는 위로
```

**왜 팩트가 위로하는가:**
- 팩트는 판단하지 않는다 → "게으른 게 아니라 전두엽이 소진된 것"
- 팩트는 보편적이다 → "모든 인간의 뇌가 그렇다" = 나만 이상한 게 아님
- 팩트는 관점을 바꾼다 → "불안이 각성과 같은 화학식" = 약점이 무기

**예시:**
- "3만 5천 번의 결정 후 전두엽이 소진된 것이다" → 자기 용서
- "불안과 각성의 화학식은 같다" → 약점의 재정의
- "95%가 어제의 복사본이고, 5%의 틈이 변화의 자리다" → 동기부여

## 2. 핵심 톤: "담백한 격조"

- 따뜻하게 닿되 과잉 없이
- 위로하되 설교하지 않음
- 감정 건드리되 물렁하지 않음
- 담백하되 가볍지 않게
- 논리적이면서 뜨겁게 각성

**한 문장 정의:**
"지친 사람 옆에 앉아서 같이 바다 보면서 말해주는 글. 단, 문장 하나하나에 무게가 있는."

### 인칭 규칙: "우리"

- "너" "나" "당신" 아닌 **"우리"**
- 말하는 사람과 듣는 사람이 동료. 같이 걸어가는 관계.
- "당신의 뇌는" → **"우리의 뇌는"** / "기억하세요" → **"우리는 기억한다"**
- 예외: 1~2문장 강조/대비용으로 "우리 자신"은 허용

## 3. 콘텐츠 도메인 시스템

### 코어 도메인 5종

| 도메인 | 팩트 소스 | 위로/동기부여 연결 |
|--------|----------|------------------|
| **뇌과학** | 편도체, 전두엽, 호르몬, 신경회로, 시냅스 | "뇌가 그렇게 설계된 것" = 자기 용서 |
| **데이터분석** | 통계, 확률, 패턴, 수치, 비율 | "숫자가 보여주는 진실" = 객관적 위로 |
| **철학** | 존재론, 인식론, 윤리, 시간, 자유의지 | "삶의 구조를 재정의" = 관점 전환 |
| **인문학** | 역사, 심리, 사회, 문화, 인간 조건 | "인간은 원래 그랬다" = 보편적 연대 |
| **컴퓨팅** | 알고리즘, 시스템, 네트워크, 데이터, 최적화 | "기계의 논리로 삶을 비추다" = 새 은유 |

### 확장 도메인 5종

| 도메인 | 팩트 소스 | 위로/동기부여 연결 |
|--------|----------|------------------|
| **물리학** | 양자역학, 상대성, 엔트로피, 파동, 장(field) | "우주의 법칙이 삶에도 적용된다" = 보편적 질서 |
| **생물학** | 진화, DNA, 세포, 생태계, 공생 | "생명은 원래 적응한다" = 변화 수용 |
| **수학** | 확률, 무한, 프랙탈, 위상, 대칭 | "수의 아름다움이 존재를 비춘다" = 구조적 경이 |
| **경제학** | 게임이론, 행동경제학, 인센티브, 균형 | "선택의 구조를 보면 자유가 보인다" = 의사결정 |
| **예술** | 미학, 창작론, 형식/내용, 수용미학 | "표현이 존재를 완성한다" = 창조적 각성 |

### 교차점 공식

**진짜 콘텐츠는 도메인 교차에서 탄생한다.** 반드시 2개 이상 도메인이 만나야 하며, **코어 도메인 1개 이상 포함** 필수.

```
[코어 × 코어]
뇌과학 × 철학    = "디폴트 모드 네트워크는 의식이 가능성을 리허설하는 것이다"
컴퓨팅 × 인문학  = "알고리즘은 하나의 변수를 최적화한다. 인간은 의미를 최적화한다"
데이터 × 동기부여 = "의식적 결정은 2%뿐이다. 그 2%가 자유의지가 사는 곳이다"
뇌과학 × 컴퓨팅  = "시냅스 가소성은 뇌의 git commit이다. 매일 다른 버전의 나"
철학 × 데이터    = "존재의 무게는 측정할 수 없다. 그래서 존재한다."
인문학 × 뇌과학  = "트라우마는 편도체의 과잉 아카이빙이다. 뇌가 너를 보호하려 한 것"

[코어 × 확장]
뇌과학 × 물리학  = "의식은 양자 현상인가. 관측이 현실을 만든다면."
컴퓨팅 × 생물학  = "유전자는 자연의 코드다. 돌연변이는 자연의 디버깅이다."
철학 × 수학      = "무한은 개념이 아니라 구조다. 존재도 그렇다."
데이터 × 경제학  = "행동경제학은 데이터로 비합리를 증명한다. 비합리가 인간을 만든다."
인문학 × 예술    = "수용미학은 관객이 작품을 완성한다고 말한다. 삶도 그렇다."
```

## 4. 글쓰기 구조: 3종 데이터 변환

**기본 구조는 3단계 변환(A). 주제에 따라 대안 구조(B, C)도 선택 가능.**

### 구조 A: 3단계 변환 (기본, 권장)

```
[Stage 1] 데이터 제시 — 차가운 팩트 ── 대본의 약 30%
  짧은 문장. 숫자. 사실. 감정 없음.
  "하루에 우리 뇌가 처리하는 생각의 수."
  "6만 개."

  SI 설계: 0.3~0.5
  비주얼: counter_up, data_bar, grid_morph, text_typewriter
  음악: 미니멀 사인파, 스파스 클릭

[Stage 2] 철학적 전환 — 데이터가 질문이 되는 순간 ── 대본의 약 40%
  "이것은 ~이 아니라 ~이다" 재정의 문장
  데이터가 존재론적 물음으로 변하는 전환점

  SI 설계: 0.6~0.8
  비주얼: particle_split_ratio, color_shift, neural_network, fractal_crack
  음악: 텐션 빌드업, 멜로디 상행/하행

[Stage 3] 실존적 각성 — 숫자 너머 ── 대본의 약 30%
  차가운 분석이 따뜻한 각성으로.
  마지막 문장은 데이터 언어가 아닌 인간의 언어.

  SI 설계: 0.7~0.9 (피크) → 0.4 (여운)
  비주얼: particle_escape, light_source, particle_chain_awaken
  음악: 클라이맥스 → 릴리스 멜로디
```

**적합**: 대부분의 주제. 팩트→질문→깨달음의 자연스러운 흐름.
**SI 커브**: 표준 내러티브, 크레셴도, 계단

### 구조 B: 역순 각성 (Hook-first)

```
[Stage 1] 각성 훅 — 가장 강한 문장으로 시작 ── 대본의 약 20%
  첫 문장이 결론. 시청자를 즉시 잡는다.
  "공포와 각성의 화학식은 같다."

  SI 설계: 0.7~0.9 (즉시 피크)
  비주얼: particle_chain_awaken, fractal_crack
  음악: 충격파 오프닝 (shockwave arc)

[Stage 2] 데이터 근거 — "왜 그런지" 팩트 뒷받침 ── 대본의 약 50%
  훅의 근거를 하나씩 풀어간다. 데이터가 훅을 증명한다.
  숫자, 메커니즘, 과학적 사실.

  SI 설계: 0.3~0.5 (의도적 하강 → 서서히 재상승)
  비주얼: counter_up, data_bar, neural_network, grid_morph
  음악: 미니멀 → 점진 빌드

[Stage 3] 철학적 착지 — 재정의/관점 전환 ── 대본의 약 30%
  데이터가 쌓인 후, 관점이 전환된다.
  처음의 훅이 다른 의미로 다가온다.

  SI 설계: 0.6→0.8 (재상승) → 0.4 (여운)
  비주얼: color_shift, light_source, particle_converge
  음악: 빌드업 → 릴리스
```

**적합**: 충격적 팩트, 반전 메시지, "이미 알고 있던 것의 재발견".
**SI 커브**: 충격파, 역전

### 구조 C: 이중 나선 (Dual Helix)

```
두 개의 데이터 스트림이 교차하며 상승한다.

[A1] 데이터 A 제시 ── ~15%
  "인간의 심장은 하루에 10만 번 뛴다."
  SI: 0.4

[B1] 데이터 B 제시 ── ~15%
  "알고리즘은 초당 10억 번 연산한다."
  SI: 0.4

[A2] 질문 A — 데이터가 의미로 ── ~20%
  "10만 번의 박동 중 우리가 느끼는 건 몇 번일까."
  SI: 0.6

[B2] 질문 B — 대조적 질문 ── ~20%
  "10억 번의 연산 중 의미를 아는 건 몇 번일까."
  SI: 0.7

[합류] 각성 — 두 스트림이 만나는 순간 ── ~30%
  "심장은 세지 않고 뛰고, 알고리즘은 세기만 한다."
  "그 차이가 존재의 무게다."
  SI: 0.8→0.9 (피크) → 0.4 (여운)
```

**적합**: 대조적 두 개념 비교 (인간 vs 기계, 공포 vs 각성, 데이터 vs 직관).
**SI 커브**: 파동 (A↑B↓ 교차 → 합류 피크)

### 문장(마디) 설계 원칙

```
1. 모든 문장은 1마디(1줄) 단위로 쪼갠다.
   [나쁜 예] 
   "하루에 우리 뇌가 처리하는 생각의 수는 무려 6만 개나 됩니다." (27자 - 2마디 오버플로우)
   [좋은 예]
   "하루에 우리 뇌가 처리하는 생각." (15자 - 1마디 꽉 채움)
   "6만 개." (4자 - 강한 악센트 + 남은 마디 비트 솔로)

2. 재정의 문장은 대비로
   "이건 습관이 아니다."
   "자동화다."

3. 드롭(Drop) 구간의 활용
   전환점 직전이나 최고조 직후에는 반드시 '빈 줄'을 넣어 음악이 솔로로 치고 나올 공간을 준다.

4. 전환점에 가장 강한 극단문 (SI 피크)
   "생존 전략이,"
   "존재를 잠식했다."

5. 대조 구조는 반드시 2세그먼트 연속 (비트 매칭)
   "오전의 가석방 승인율. 65퍼센트."
   "오후 늦게의 가석방 승인율. 7퍼센트."
```

## 5. 시맨틱 인텐시티(SI) 안무 가이드

대본의 모든 문장은 시스템에 의해 SI(의미 강도, 0~1)로 변환된다.
**SI가 비주얼 이펙트 강도, 음악 텍스처 밀도, TTS 렌더링을 결정한다.**
따라서 작가는 SI를 **의도적으로 설계**해야 한다.

### SI 계산 공식 (시스템 실제값)

```
SI = verb_score × 0.30 + emotion_score × 0.30 + structure_score × 0.20 + density_score × 0.20
```

- **verb_score**: 동사의 에너지 (폭발=0.9, 흐르다=0.45, 있다=0.1)
- **emotion_score**: 감정어 (공포=0.9, 각성=0.75, 평온=0.1)
- **structure_score**: 문장 길이 (≤15자=+0.3), 느낌표(+0.15), 다중문장(+0.15)
- **density_score**: 데이터 밀도 (숫자, 화학물질, 신체부위, 과학용어 = 각 +0.1)

### SI 레벨별 글쓰기 전략

| SI | 문장 설계법 | 시스템 반응 |
|----|-----------|-----------|
| **0.1~0.3** (고요) | 긴 문장(25자+), 저에너지 동사(있다/되다/흐르다), 데이터 없음 | 비주얼 최소, 음악 스파스 사인파 |
| **0.4~0.5** (전개) | 중간 문장, 중에너지 동사(움직이다/퍼지다), 과학 용어 1개 | TTS 폰트 성장, 스캔라인 시작 |
| **0.6~0.7** (고조) | 짧은 문장(15~25자), 고에너지 동사(깨우다/상승/돌파), 감정어 1개 | 글리치 블록, 크로매틱, 음악 텐션 빌드 |
| **0.8~0.9** (폭발) | 초단문(≤15자), 최고 동사 + 감정어 + 데이터 3요소 동시 | 풀 이펙트, 폰트 48px, 피드백, 유클리드 리듬 |

### SI 커브 템플릿 (7종)

```
표준 내러티브: Hook(0.5) → Setup(0.3) → Build(0.5→0.7) → Peak(0.9) → Resolution(0.4)
크레셴도:      Low(0.2) → Slow build(0.3→0.5→0.7→0.9) → Hold(0.8)
역전:         High(0.7) → Drop(0.3) → Rebuild(0.5→0.9) → Fade(0.3)
파동:         Mid(0.5) → Peak1(0.7) → Valley(0.3) → Peak2(0.9) → Fade(0.4)
```

## 7. 음악-글쓰기 연동 가이드

대본의 감정 태그와 SI가 ikeda 음악을 직접 결정한다:

### 감정 → 멜로디 매핑

- curious/analytical 문장 → **ascending 멜로디** (음 올라감, 탐구)
- tension/warning 문장 → **descending 멜로디** (음 내려감, 위기)
- fear/crisis 문장 → **tension 시퀀스** (비팅 가속, 불안 증폭)
- resolution/hope 문장 → **release 시퀀스** (비팅 감속, 해소)

### SI → 텍스처 밀도

| SI 범위 | 음악 텍스처 |
|---------|-----------|
| < 0.3 | 스파스 사인파 + 미니멀 클릭 |
| 0.3~0.5 | 사인 멜로디 + data_click + 라이트 리듬 |
| 0.5~0.7 | 풀 사인 멜로디 + 리듬 백본(킥/하이햇) + bytebeat 텍스처 |
| > 0.7 | 모든 텍스처 활성 + 피드백 + 유클리드 리듬 |

**글쓰기 팁**: 감정 전환이 급격하면 음악도 급변한다. 자연스러운 감정 곡선을 설계하면 음악도 자연스럽다.

## 8. 마디 조형 (Sentence Sculpting) 규칙

### 마디 호흡 (Rhythm & Flow) 원칙
- **산문처럼 쓰지 않는다. 운문처럼 조각한다.**
- 문장을 끝까지 설명하려 하지 않는다. 명사형 종결, 짧은 단어의 던짐(Drop)을 적극 활용한다.
- "근데" → "하지만" / "그런데"
- "느꼈다" → "원망", "외면" (명사형 압축 권장)

### 어휘 레벨 기준
- **허용:** 사유, 해독, 역학, 구조, 문법, 원초적, 탐구, 본질, 작동 원리, 루프, 자동화, 균열, 잠식, 수렴, 발산, 최적화, 시뮬레이션
- **지향:** 일상에서 이해 가능하되 한 층 위의 어휘
- **금지:** 학술 논문체, 번역투, 과시적 한자어 남용

### 반복 어휘 경계
- 한 마디의 반복 구조(Rhyme)를 의도한 것이 아니라면, 불필요한 조사/접속사 남용은 피한다.
- 동일한 논지를 다른 표현으로 반복하는 것은 설명이지 리듬이 아니다.

### SI 최적화 규칙 (v12 추가)
- **절대 원칙**: 한 줄거리(1줄)에 **20자 초과 금지**. 불가피할 경우 2줄로 쪼갠다.
- 전환점(Build-up → Drop)에는 시각적 공백(빈 줄)을 넣어 음악이 솔로 연주되도록 한다.
- 대조 구조는 **반드시 2세그먼트 연속** (시스템이 패턴으로 인식)
- Stage 2 전환점에 **재정의 문장** 반드시 1개 이상
- 피크(Drop 직후) 마디는 **SI 0.7 이상** 목표

### 허용/금지 표현

**허용:**
- 감정에 닿는 문장, 공감 기반 질문, 논리적 각성
- 데이터 직접 제시 ("6만 개", "95퍼센트")
- 과학/기술 용어의 철학적 전용 ("루프", "자동화", "수렴")
- 비주얼이 보이는 동사 ("잠식하다", "이탈하다", "수렴하다")

**금지:**
- 싸구려 감정 호소, 과잉 수사, 클리셰
- 설교조, 뻔한 비유, 선배 꼰대 느낌
- **가짜로 깊은 척 — 유일한 금기**
- 데이터 출처 나열 ("연구에 따르면")
- 데이터 해석을 설명조로 풀기 ("이것은 ~를 의미합니다")

⚠️ 이 단계의 핵심 팩트는 아직 미검증 상태. STEP 0.5에서 검증한다.

**이전 에피소드 주제와 겹치지 않을 것:**
- EP001: 뇌의 복사 패턴 (95% 반복)
- EP002: 오답의 학습 효과
- EP003: (완료)
- EP004: 의사결정 소진 (Decision Fatigue)
- EP005: 공포와 각성의 화학식
- EP006: 선택의 중력 (실패, 참고 금지)
- EP007: 이진탐색 × 덜어냄 × 나아감
- EP008: 엔트로피 × 카오스몽키 (실패, 참고 금지)

**추천 형식:**
```
1. "우리의 기억은 몇 번이나 조작됐을까"
   뇌과학 × 철학 | 팩트: 기억 인출 시 해마가 매번 재구성
   이유: 기억의 불완전성이 오히려 생존 전략이라는 반전

2. "알고리즘이 당신보다 당신을 잘 아는 이유"
   컴퓨팅 × 인문학 | 팩트: 추천 시스템 정확도 vs 자기인식
   이유: 데이터가 보여주는 자기이해의 한계와 가능성
```

### STEP 0.5: ★ 팩트 검증 게이트 ★ (주제 선택 후, 글쓰기 전)

사용자가 주제를 선택하면, **대본 작성 전에** 핵심 팩트를 검증한다.

**절차:**
1. 대본에 사용할 핵심 팩트 3~7개를 목록으로 나열
2. 각 팩트를 **웹서치로 검증** (WebSearch 도구 사용 필수)
3. 검증 결과를 사용자에게 보고

**보고 형식:**
```
🔍 팩트 검증 결과

| # | 팩트 | 판정 | 근거 |
|---|------|------|------|
| 1 | "하루 500억 개 세포 사멸" | ✅ | 출처: ... |
| 2 | "신생아 뇌세포 > 성인" | ❌ | 뉴런 수는 비슷. 줄어드는 건 시냅스 |
| 3 | "시냅스 1000조 개" | ⚡ | 과장. 100~500조가 학계 추정치 |

✅ = 검증됨 | ⚡ = 근사치/애매 (수정 필요) | ❌ = 오류 (사용 금지)
```

**게이트 규칙:**
- ❌ 팩트가 1개라도 있으면 → 해당 팩트를 수정하거나 제거한 후 진행
- ⚡ 팩트는 → 안전한 표현으로 수정 방안을 함께 제시
- **모든 팩트가 ✅ 또는 ⚡(수정안 확보) 상태여야 STEP 1 진행 가능**
- 검증 없이 STEP 1 진행 절대 금지

**검증 기준:**
- 구체적 숫자: 반드시 출처 확인. 출처 불분명하면 ⚡ 처리 후 "약/수백/수만" 등 안전 표현 제시
- 과학적 메커니즘: 핵심 인과관계가 맞는지 확인. 대중적 오해와 학술적 사실 구분
- 인용/일화: 실제 기록된 출처 여부 확인. 출처 불분명하면 "~로 전해진다" 표현 권장
- 비유적 사용: 팩트를 은유로 쓸 때도 원본 팩트 자체는 정확해야 함

### STEP 1: 대본 작성 + 제출

- **STEP 0.5의 검증된 팩트만 사용** (검증 안 된 새 팩트 추가 금지. 추가 시 재검증 필수)
- 대본 본문만 제출
- **시스템 최적화 점수를 자동 첨부** (섹션 12 참조)
- 제목 후보, 음악 제안 등 일체 포함하지 않음
- "대본을 확인해주세요. 수정할 부분이 있으면 말씀해주세요." 로 마무리

### STEP 2: ★ 글 컨펌 게이트 ★

사용자의 명시적 승인을 기다린다.

| 사용자 발화 | 판단 | 행동 |
|------------|------|------|
| "좋아", "컨펌", "OK", "확정" | 컨펌 | STEP 3 진행 |
| "이걸로 가자", "진행해" | 컨펌 | STEP 3 진행 |
| "여기 수정해줘", "~부분 바꿔" | 수정 | 수정 후 STEP 1 재제출 (재채점) |
| "음... 괜찮은데" (애매) | 미확정 | "확정해도 될까요?" 재확인 |
| "다시 써줘", "다른 방향으로" | 재작성 | STEP 0 or 1로 복귀 |
| (아무 응답 없이 다른 질문) | 미확정 | 글 컨펌 여부 먼저 확인 |

⚠️ **사용자가 컨펌하지 않았다면:**
- 제목을 물어보지 않는다
- 음악을 물어보지 않는다
- 어떤 제작 단계도 시작하지 않는다
- 그냥 기다린다
