## 설계 원칙: 제네레이티브 → 리액티브 2레이어

```
대본 (완성물)
 │
 ├─ 스토리 구조 (서사, 전환점, 문단 흐름)
 │   → 음악 뼈대 설계        [제네레이티브]
 │   → 비주얼 형상/장면 설계  [제네레이티브]
 │
 └─ SI (감정 강도, 0.05~1.0)
     → 음악 표현 변조 (음색, 드라이브)  [리액티브]
     → 비주얼 반응 (밝기, 레이어 수)    [리액티브]
     → audio → 비주얼 프레임 반응       [리액티브] ← 이미 강함, 유지
```

### 왜 이 구조인가

| 문제 | 원인 | 해결 |
|------|------|------|
| 음악이 대본과 약하게 연결됨 | SI 하나로 모든 걸 처리 (볼륨 ±5%) | 스토리→구조 설계 + SI→표현 변조, 역할 분리 |
| SI가 음악 완성도를 해침 | 감정 데이터가 음악 구조에 직접 간섭 | SI는 표면(음색/드라이브)만. 구조는 건드리지 않음 |
| 비주얼이 추상적 | 리액티브만 있고 형상 생성 없음 | 스토리→형상 생성(제네레이티브) + audio→반응(리액티브) |

### 핵심 규칙

1. **제네레이티브가 먼저**: 스토리 구조가 음악/비주얼의 뼈대를 설계한다
2. **리액티브는 표면만**: SI/audio는 완성된 구조를 해치지 않고 표현만 변조한다
3. **음악의 자기 문법 보호**: 뼈대 안에서 음악은 4바/8바 문법으로 자유롭게 완성된다

## 현재 상태 진단

### 세 경로의 현실 (상세: `docs/reactive-architecture.md`)

| 경로 | 체감도 | 문제 |
|------|--------|------|
| 대본→음악 | **약** | SI가 볼륨 변조에 집중. 스토리 구조→편곡 연결 없음 |
| 대본→비주얼 | **중** → 강 목표 | 전략 승격은 작동. 형상 생성 부재 |
| 음악→비주얼 | **강** | 프레임 단위 반응. 유지 |

### 대본에서 추출 가능한 데이터 (현재 vs 목표)

| 데이터 | 현재 | 목표 역할 |
|--------|------|----------|
| **SI** (감정 강도) | 음악 볼륨 + 비주얼 전략 승격 | 리액티브 전용 — 음색/드라이브/밝기 변조 |
| **스토리 구조** (문단/전환점/주제 흐름) | 미추출 | 제네레이티브 — 편곡/섹션/형상/장면 설계 |
| **키워드/품사** (kiwipiepy) | 비주얼 vocab 매핑 | 유지 + 확장 가능 |

## 목표 1: 스토리→음악 제네레이티브

### 현재 → 목표

```
현재: ep_seed + 장르 스펙 → ARRANGEMENT_TABLE (고정)
     SI → 볼륨 ±5%, 게이트, 음색 (미약한 리액티브)

목표: 스토리 구조 → ARRANGEMENT_TABLE 동적 생성 (제네레이티브)
     SI → 음색/드라이브 변조만 (리액티브, 구조 비간섭)
```

### 스토리가 결정할 수 있는 것 (후보)

- 섹션 전환 시점 (문단 경계 → 가장 가까운 4바 snap)
- 레이어 ON/OFF 시점 (스토리 에너지 곡선 기반)
- 에너지 아크 (intro→build→climax→resolution 매핑)
- 장르 내 변주 방향 (ep_seed와 협력)

### 스토리가 결정하지 않는 것

- BPM (장르 범위 내 고정)
- 드럼 패턴 (seq_config)
- 멜로디 음정 (Norgard 수열 + ep_seed)
- 음색 디테일 (SI 리액티브 담당)

## 목표 2: 스토리→비주얼 제네레이티브

### 현재 → 목표

```
현재: SI → 전략 승격, 레이어 수 (구조적 리액티브)
     키워드 → vocab 매핑 (이미 작동)
     결과: 추상 패턴만. 인식 가능한 형상 없음

목표: 스토리 → 형상/장면 생성 (제네레이티브)
     SI + audio → 생성된 형상이 반응 (리액티브)
```

### 레퍼런스와의 갭

| 레퍼런스 | 현재 | 필요한 것 |
|---------|------|----------|
| 픽셀 스프라이트 (토끼, 아이콘) | pixel_grid = 추상 패턴 | 비트맵 형상 사전 |
| ASCII 실루엣 (인체+코드) | ascii_block = 키워드 텍스트 | 마스크 기반 텍스트 채움 |
| 컬러 ASCII 초상 | 없음 | 이미지→ASCII 변환 |
| 와이어프레임 구체 | terra_globe (거의 일치) | 스타일 조정만 |
| 대형 ASCII 타이포 | ascii_block + text_reveal (거의 일치) | 크기/배치 강화 |

### 구현 방향 (미확정)

- 기존 스택 확장 우선: Python(Pillow) + React(Canvas 2D) + Three.js
- 비용 0원 원칙 유지
