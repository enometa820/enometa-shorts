# ENOMETA 시스템 창작 자유도 감사 (Creative Freedom Audit)

> 작성일: 2026-03-04
> 성격: 점검 보고서 (코드/규칙 수정 없음, 향후 참조용)

## Context

ENOMETA의 본질: **"매 에피소드가 시스템을 성장시킨다 — 같은 패턴 반복 금지"**

하지만 EP005~EP007 피드백이 누적되면서, "안전성"을 위한 규칙들이 점점 창작 공간을 좁히고 있다. 시스템이 "나쁜 결과를 방지"하려다 "새로운 시도 자체를 방지"하는 상태가 되었다.

이 감사의 목적: **시스템의 본질(무한한 창작 가능성)을 저해하는 불필요한 리미트를 식별하고, "절대 규칙"과 "가이드라인"을 구분**하는 것.

---

## 발견된 제약: 심각도별 분류

### TIER 1: 창작 공간을 구조적으로 봉쇄하는 제약

#### 1-1. 글쓰기 구조 "유일한 구조" 선언
- **위치**: `skills/enometa-writing.md` 섹션 4, 라인 88
- **현재**: `"모든 대본은 이 3단계를 거친다. 유일한 구조."`
- **문제**: 순환적, 병렬적, 역순, 변증법적, 2단계, 4단계 등 모든 대안적 내러티브 구조를 원천 봉쇄
- **채점 시스템이 강제**: 시스템 최적화 점수에서 "3단계 전환" 15점 배점 → 따르지 않으면 자동 감점
- **모순**: "매 에피소드가 시스템을 성장시킨다"면서 구조는 영원히 고정

#### 1-2. ikeda 단일 장르 강제
- **위치**: `scripts/enometa_music_engine.py` 라인 3036~3076, `CLAUDE.md` 라인 9
- **현재**: 다른 장르 선택 시 자동으로 ikeda로 덮어씀
- **문제**:
  - BPM 108~162 (±20%) 범위만 가능 — 느린/빠른 음악 불가
  - Minor 키 7개만 — Major 키 완전 배제
  - Euclidean 리듬 고정 — swing, polyrhythm 등 불가
  - melodic_sequence 의무 포함 — 순수 리듬/노이즈 실험 불가
- **MEMORY.md의 자기기만**: "ikeda 다양성 점검 결과: 제약 아님" — 실제로는 BPM, 키, 리듬 모두 제약

#### 1-3. SI 모듈레이션의 사실상 무효화
- **위치**: `enometa_music_engine.py` 라인 2145
- **현재**: `si_modulation = 0.95 + si_env * 0.1` (범위 0.95~1.05 = **10% 변화폭**)
- **이전**: `0.7 + si * 0.6` (범위 0.7~1.3 = **86% 변화폭**)
- **문제**: SI가 0이든 1이든 음악이 거의 같음 → 조용한 구간과 강렬한 구간의 대비 소멸
- **EP006 피드백 과잉 보정**: "음악이 너무 조용했다" → 전구간을 공격적으로 밀어붙이는 것으로 해결 → 다이나믹 레인지 자체를 제거

#### 1-4. 감정 → 비주얼 고정 매핑
- **위치**: `scripts/visual_script_generator.py` 라인 287~365 (`EMOTION_VOCAB_POOL`)
- **현재**: 11개 감정 × 각 3+3개 vocab = 고정된 풀에서만 선택
- **문제**:
  - tension → 항상 glitch 텍스트모드 (부드러운 tension 표현 불가)
  - awakening → 항상 scatter 텍스트모드 (다른 방식의 각성 불가)
  - bg_mode가 "calm"/"turbulent" 2개뿐
  - 같은 감정을 다른 비주얼로 표현하는 경로가 없음

### TIER 2: 다양성의 천장을 만드는 제약

#### 2-1. 피드백 규칙의 "절대화" 문제
- **위치**: `scripts/feedback_defaults.json` 전체
- **패턴**: EP005/EP006/EP007 피드백이 "반드시/금지/절대" 규칙으로 고착
- **예시들**:
  - `"hybrid 모드에서도 Remotion vocab을 Python 배경 위에 반드시 오버레이"` → Pure Python 비주얼 불가
  - `"TextReveal position='bottom' 금지"` → 상황에 따라 하단이 적합할 수도 있음
  - `"순수 노이즈 금지"` → 노이즈 아트 장르 완전 배제
  - `"bytebeat: 최소화"`, `"feedback: 최소화"` → 글리치/디지털 음악 억제
- **구조적 문제**: 규칙 제거/완화 메커니즘이 없음 → 에피소드가 쌓일수록 규칙만 누적

#### 2-2. SI 게이트 최소값 0.45 강제
- **위치**: `enometa_music_engine.py` 라인 1006~1027, `feedback_defaults.json` 라인 115
- **현재**: SI=0일 때도 45% 이상 악기 볼륨 유지
- **문제**: 침묵, 미니멀리즘, 극단적 다이나믹 컨트래스트 불가 → 음악이 "쉬는" 순간이 없음

#### 2-3. 레이어 수의 SI 완전 종속
- **위치**: `visual_script_generator.py` `build_scene()` 함수
- **현재**: `si_max_layers = max(1, min(3, int(si * 3.5)))` — SI 낮으면 무조건 1레이어
- **문제**: "조용하지만 시각적으로 풍부한" 씬 불가능. SI가 의미 강도인데, 비주얼 밀도와 항상 비례해야 할 이유가 없음

#### 2-4. Python 레이어 프리셋 고정
- **위치**: `visual_renderer.py` 라인 101~114 (`GENRE_LAYER_PRESETS["ikeda"]`)
- **현재**: 음악 레이어 3개(강도 0.7/0.4/0.3) + TTS 레이어 4개(0.95/0.85/0.75/0.65) 절대 고정
- **문제**: 새로운 레이어 조합, 강도 조절 불가 → 모든 에피소드의 Python 배경이 같은 느낌

#### 2-5. ShapeMotion 감정별 패턴 완전 고정
- **위치**: `src/ShapeMotion.tsx` 라인 224~241
- **현재**: tension=회전사각형, climax=동심원, awakening=스캔라인, intro=부유점, buildup=삼각형 (5개만)
- **문제**: 각 패턴의 속도/크기/개수도 하드코딩 (회전 45도/초 고정, 펄스 주파수 3 고정)

#### 2-6. 아르페지오 패턴 5개 고정
- **위치**: `enometa_music_engine.py` 라인 3097~3103
- **현재**: 6음짜리 패턴 5개만 (all same length)
- **문제**: 10+ 에피소드에서 반복 불가피. 3음, 8음, 비대칭 패턴 등 변형 불가

#### 2-7. 음량 스케일 하드코딩
- **위치**: `enometa_music_engine.py` 라인 3044~3063
- **현재**: `feedback: 0.0`, `fm_bass: 0`, `synth_lead: 0`, `bytebeat: 0.15`
- **문제**: 볼륨이 0인 악기는 force_active/EMOTION_MAP에서 켜도 소리 안 남 → 사실상 삭제된 악기

## 근본 원인 분석

### 1. "안전성 우선" 철학의 과잉
EP005(5/10), EP006("완전 실패") 이후 "다시는 실패하지 않겠다"는 방어적 코딩이 시스템 전체를 지배. EP007(98/100)의 성공이 "이것이 정답"이라는 신화로 고착.

### 2. 피드백 규칙의 단방향 누적
규칙은 추가만 되고 제거/완화 메커니즘 없음. 에피소드가 쌓일수록 자유도 감소.

### 3. "절대 규칙"과 "가이드라인"의 혼동
기술적 필수사항(`py` 명령, Edge-TTS)과 창작적 선택(장르, 구조)이 같은 "절대 규칙" 레벨.

### 4. MEMORY.md의 자기 확증
"ikeda 다양성: 제약 아님" — 실제 코드와 불일치하는 결론이 메모리에 고착.

## "절대 규칙" vs "가이드라인" 재분류 제안

### 진짜 절대 규칙 (유지)
| 규칙 | 근거 |
|------|------|
| `py` 명령 사용 | Windows Store alias 기술 문제 |
| Edge-TTS 전용 | Chatterbox 품질 문제 |
| `useCurrentFrame()` 기반 애니메이션 | Remotion 렌더링 원리 |
| 대본 컨펌 게이트 | 워크플로우 안전장치 |
| `seededRand()` 사용 | Remotion 결정론 |
| **비주얼 해상도 1:1 (1080x1080)** | **미적 절대규칙 (사용자 결정)** |

### "절대"에서 "기본값(default)"으로 완화 후보
| 현재 절대 규칙 | 완화 방향 |
|--------------|----------|
| 3단계 구조 "유일한 구조" | 기본 구조이되, 대안 구조 허용 |
| ikeda 단일 장르 | 기본 장르이되, BPM/키/리듬 범위 확대 |
| SI 모듈레이션 0.95~1.05 | 다이나믹 레인지 복원 |
| si_gate 최소 0.45 | 음악의 "숨쉬기" 허용 |
| hybrid 모드 강제 | 기본 모드이되, 변형 가능 |
| narration=0.90, bgm=1.0 고정 | 기본값이되, 에피소드별 조정 가능 |
| 감정→vocab 고정 매핑 | 기본 매핑이되, override 파라미터 지원 |
| feedback_defaults의 "반드시/금지" | "기본값/권장" 용어로 전환 |
