> 새 개념이 등장할 때마다 여기에 추가된다

## Git & 버전 관리

| 용어 (영어) | 발음 | 한 줄 설명 |
|-------------|------|-----------|
| **Git Hooks** | 깃 훅 | git 명령 실행 전/후에 자동으로 스크립트를 끼워 넣는 장치. pre-commit은 커밋 직전에 실행 |
| **Repository Hygiene** | 리포지토리 하이진 | 저장소에 불필요한 파일이 쌓이지 않도록 관리하는 습관. untracked 파일 정리, .gitignore 관리 등 |
| **Staging Area** | 스테이징 에어리어 | `git add`로 커밋할 파일을 올려두는 대기 공간. 커밋 전 "무엇을 담을지" 선택하는 단계 |
| **Commit Prefix** | 커밋 프리픽스 | `feat:` `fix:` `docs:` 같이 커밋 메시지 앞에 붙이는 약속된 태그. 변경 성격을 한눈에 표시 |
| **Untracked Files** | 언트랙드 파일즈 | git이 아직 추적하지 않는 새 파일. `git status`에서 빨간색으로 표시됨 |

## 개발 도구 & 워크플로우

| 용어 (영어) | 발음 | 한 줄 설명 |
|-------------|------|-----------|
| **DX (Developer Experience)** | 디엑스 / 개발자 경험 | 개발자가 작업하는 환경과 흐름이 얼마나 편리한지. 좋은 DX = 실수가 줄고 속도가 올라감 |
| **Convention Enforcement** | 컨벤션 인포스먼트 | 팀 규칙을 사람이 아닌 코드/도구가 강제하는 것. 실수를 막는 자동화된 안전장치 |
| **Configuration Management** | 컨피규레이션 매니지먼트 | 도구의 동작 방식을 코드(설정 파일)로 정의하는 것. 사람마다 다르게 설정하는 혼란을 방지 |
| **Non-interactive CLI** | 논-인터랙티브 씨엘아이 | 사람 입력(input()) 없이 인수(argument)만으로 실행되는 명령줄 도구. 자동화/스크립트에 필수 |
| **Pipeline** | 파이프라인 | 작업이 단계별로 이어지는 흐름. A → B → C처럼 앞 단계 결과가 다음 단계 입력이 됨 |
| **Workflow Documentation** | 워크플로우 도큐멘테이션 | 작업 흐름을 문서화하거나 자동화하는 것. ENOMETA 스킬 파일이 이 역할 |

## 아키텍처 & 설계

| 용어 (영어) | 발음 | 한 줄 설명 |
|-------------|------|-----------|
| **Trade-off** | 트레이드오프 | 하나를 얻으면 다른 하나를 잃는 상충 관계. 예: 단순함 vs 유연함 |
| **Entry Point** | 엔트리 포인트 | 프로그램이 시작되는 지점. ENOMETA에서는 `src/index.tsx` |
| **Monolith** | 모놀리스 | 모든 기능이 하나의 큰 파일/코드에 몰려있는 구조. 장점: 단순함. 단점: 수정 시 전체 영향 |
| **ADR (Architecture Decision Record)** | 에이디알 | 아키텍처 결정을 문서로 남기는 형식. 왜 이렇게 선택했는지 이유를 기록. `docs/decisions/` 폴더 |
| **Hybrid** | 하이브리드 | 두 가지 방식을 섞은 것. ENOMETA에서는 Python(프레임 생성) + Remotion(합성) 조합 |

## Python & 스크립트

| 용어 (영어) | 발음 | 한 줄 설명 |
|-------------|------|-----------|
| **CLI Arguments** | 씨엘아이 아규먼츠 | 명령줄에서 프로그램에 넘기는 값. `--title "제목"` 같은 것. argparse로 처리 |
| **argparse** | 아규파스 | Python에서 CLI 인수를 파싱(해석)하는 표준 라이브러리 |
| **Exit Code** | 엑싯 코드 | 프로그램 종료 시 반환하는 숫자. 0=성공, 1=오류, 2=특수목적(hook 블록 등) |
| **Environment Variable** | 환경 변수 | 프로세스 실행 환경에 설정된 변수. `os.environ.get("KEY")` 로 읽음. CLAUDE_TOOL_INPUT이 이것 |
| **Morpheme Analysis** | 모피엠 어낼리시스 | 형태소 분석. 한국어 텍스트를 의미 단위(명사/동사 등)로 쪼개는 작업. kiwipiepy가 이것 |

## 오디오 & 미디어

| 용어 (영어) | 발음 | 한 줄 설명 |
|-------------|------|-----------|
| **LUFS (Loudness Units Full Scale)** | 루프스 | 음악의 라우드니스(체감 음량) 단위. YouTube 기준 -14 LUFS. 숫자가 클수록(0에 가까울수록) 큼 |
| **Sidechain** | 사이드체인 | 한 신호(킥)가 다른 신호(BGM)의 볼륨을 눌러주는 기법. 펌핑 효과 |
| **TTS (Text-to-Speech)** | 티티에스 | 텍스트를 음성으로 변환하는 기술. ENOMETA는 Edge-TTS 사용 |
| **BPM (Beats Per Minute)** | 비피엠 | 분당 박자 수. 음악 속도 단위. 108~162 범위가 ENOMETA 기준 |
| **Reactive** | 리액티브 | 어떤 값에 반응해서 실시간으로 변하는 것. "오디오 리액티브" = 음악 신호에 반응해 비주얼이 변함 |
